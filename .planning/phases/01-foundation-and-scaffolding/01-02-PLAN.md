Step 1 -- Create `src/proxy.ts` with Clerk v7 middleware protecting all `/admin/*` routes. Use the exact pattern from 01-RESEARCH.md Pattern 1:

- Import `clerkMiddleware` and `createRouteMatcher` from `@clerk/nextjs/server`
- Import `NextResponse` from `next/server`
- Define `isAdminRoute` matcher for `/admin(.*)`
- Define `withSecurityHeaders()` function adding: X-Content-Type-Options: nosniff, X-Frame-Options: SAMEORIGIN, Referrer-Policy: strict-origin-when-cross-origin, X-XSS-Protection: 1; mode=block, Permissions-Policy: camera=(), microphone=(), geolocation=(), Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
- Export default `clerkMiddleware(async (auth, req) => { ... })`:
  - If admin route: get userId and sessionClaims via `await auth()`
  - If no userId: redirect to `/sign-in?redirect_url=/admin`
  - If role !== "admin" (from sessionClaims.metadata.role): redirect to `/`
  - Always apply security headers to response
- Export `config` with matcher array (same as 01-RESEARCH.md)

**IMPORTANT:** If Clerk v7 has type issues or import errors, fall back to `@clerk/nextjs@^6.39.0`. If the `proxy.ts` file convention causes issues, rename to `middleware.ts`. Document which approach was used in the SUMMARY.
